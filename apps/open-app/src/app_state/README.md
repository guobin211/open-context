# AppState

应用状态管理模块，负责管理应用运行时的各种状态数据。

## 数据结构

### 原子类型

用于建立知识库和搜索索引的基础数据单元：

| 类型           | 说明                             |
| -------------- | -------------------------------- |
| `file`         | 文件对象                         |
| `folder`       | 文件夹对象                       |
| `link`         | 网页地址                         |
| `note`         | 笔记对象                         |
| `repo`         | 代码仓库对象                     |
| `conversation` | 会话对象，包含参与者、消息记录等 |

用于应用界面和交互的数据单元：

| 类型       | 说明                               |
| ---------- | ---------------------------------- |
| `terminal` | 终端会话，包含命令历史、当前目录等 |
| `webview`  | 网页视图，包含 URL、加载状态等     |
| `chat`     | 聊天对象，包含多个会话             |

### 复合类型

| 类型        | 说明                                 |
| ----------- | ------------------------------------ |
| `workspace` | 工作区对象，可包含多个原子类型的组合 |

## 设计原则

- 原子类型保持独立，便于索引和检索
- 复合类型通过组合原子类型实现复杂场景
- 所有数据结构支持序列化和持久化
